# Правила разработки фичи

При получении задачи на новую фичу или значительное UI-изменение, следуй этому процессу:

## Этап 1 — Параллельный запуск агентов

Запусти одновременно (в одном сообщении) следующих агентов:

1. **designer** — прорабатывает визуальный дизайн фичи, задаёт пользователю вопросы о внешнем виде, согласовывает детали
2. **Explore** — исследует кодовую базу: находит связанные файлы, существующие паттерны, модели данных, роутинг

Дождись результатов обоих агентов перед переходом к следующему этапу.

## Этап 2 — Планирование

Войди в план-режим (`EnterPlanMode`) и составь план реализации на основе:
- Согласованного дизайна от агента designer
- Найденных паттернов и файлов от агента Explore
- Правил из `AGENTS.md` (локализация, DI, структура виджетов, proto)

План должен содержать:
- Список файлов для создания/изменения
- Порядок реализации (модели → bloc → UI → тесты)
- Какие зависимости нужны (репозитории, хранилища, proto-модели)

## Этап 3 — Реализация

После одобрения плана пользователем, реализуй фичу в таком порядке:

1. **Модели и данные** — domain models, repository interfaces, data layer
2. **Бизнес-логика** — Bloc/Event/State
3. **UI** — страница и виджеты по согласованному дизайну
4. **Локализация** — строки в `app_ru.arb` + `fvm flutter gen-l10n`
5. **Роутинг** — добавить маршрут в `router.dart`
6. **Проверка** — `fvm flutter analyze`

## Этап 4 — Финализация

- Запусти golden-тесты для новых страниц (скилл `/golden-test`)
- Убедись что `fvm flutter analyze` проходит без ошибок
