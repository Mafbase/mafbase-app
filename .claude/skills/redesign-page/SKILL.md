# Redesign Page in Figma

Создай редизайн страницы `$ARGUMENTS` в Figma, опираясь на дизайн-систему из Figma.

## Figma Design System

- **Файл:** `https://www.figma.com/design/LkLr8e2DCFKQyLlnBjaTTw`
- **Страница с компонентами и токенами:** `node-id=8-2`
- **Figma — единственный источник правды** для цветов, типографики, компонентов и стилей

## Пошаговый процесс

### Шаг 1 — Анализ текущей реализации

Прочитай файл страницы `$ARGUMENTS` и определи:

1. **Назначение страницы** — что она делает, какую задачу решает
2. **Структура UI** — из каких секций/блоков состоит (AppBar, списки, карточки, формы, кнопки)
3. **Данные** — какие модели отображаются, какие поля выводятся на экран
4. **Состояния** — loading, error, empty, data (прочитай связанный Bloc/State)
5. **Действия пользователя** — кнопки, навигация, формы, диалоги
6. **Зависимости** — какие виджеты из `lib/common/widgets/` используются

Если страница использует Bloc, прочитай также:
- Файл State-класса — чтобы понять все возможные состояния
- Файл Event-класса — чтобы понять все пользовательские действия

### Шаг 2 — Получить дизайн-систему из Figma

Вызови `get_design_context` для файла дизайн-системы:
- `fileKey`: `LkLr8e2DCFKQyLlnBjaTTw`
- `nodeId`: `8:2`

Это даст тебе актуальные компоненты, цвета, типографику и стили. **Используй именно эти значения** — не хардкодь цвета и размеры, а бери их из ответа Figma.

Запиши в контекст все полученные из Figma значения:
- Палитру цветов (названия токенов и их hex-значения)
- Текстовые стили (размеры, веса, цвета)
- Параметры компонентов (скругления, отступы, тени)

### Шаг 3 — Спросить пользователя о предпочтениях

Задай пользователю вопросы через `AskUserQuestion`:

1. **Что изменить в дизайне?** — полный редизайн / только стилизация под новые токены / конкретные улучшения
2. **Какие состояния отрисовать?** — все (loading, error, empty, data) / только основное с данными
3. **Адаптивность** — desktop только / desktop + mobile
4. **Куда добавить дизайн?** — новая страница в Figma-файле / существующая страница (указать nodeId)

### Шаг 4 — Создать HTML-макет

Создай HTML-файл `docs/redesign_<page_name>.html` с редизайном страницы.

#### Правила вёрстки

1. **Цвета — ТОЛЬКО из Figma.** Возьми все цвета из результата `get_design_context` (Шаг 2). Определи их как CSS-переменные в `:root`. НЕ придумывай цвета — если нужного цвета нет в дизайн-системе, укажи это пользователю и предложи добавить его в Figma-файл дизайн-системы.

2. **Типографика — из Figma.** Шрифты, размеры и веса бери из результата `get_design_context`. Подключи нужный шрифт через Google Fonts.

3. **Компоненты — в стиле дизайн-системы из Figma.** Воспроизводи стили компонентов (кнопки, поля, карточки, диалоги) точно так, как они определены в Figma.

4. **Каждое состояние** — в отдельной секции с заголовком (например: "Loading State", "Error State", "Data State")

5. **Реалистичные данные** — заполни макет правдоподобными данными на русском языке, соответствующими модели данных страницы

6. **Иконки — ТОЛЬКО inline SVG.** НЕ используй icon fonts (Material Icons, Font Awesome и т.д.) — Figma capture не рендерит веб-шрифты и иконки превращаются в текст. Вместо этого:
   - Определи все нужные иконки как `<symbol>` внутри скрытого `<svg>` в начале `<body>`
   - Используй `<svg><use href="#icon-name"/></svg>` для вставки иконок
   - Бери SVG paths из Material Design Icons (Google Fonts)
   - Оборачивай в `<span class="icon">` для единообразного размера

7. **Добавь скрипт захвата Figma:**
   ```html
   <script src="https://mcp.figma.com/mcp/html-to-design/capture.js" async></script>
   ```

### Шаг 5 — Предпоказ пользователю

1. Запусти локальный HTTP-сервер:
   ```bash
   python3 -m http.server 8765 --directory docs &
   ```

2. Открой HTML в браузере **без** hash-параметров захвата — просто для предпросмотра:
   ```bash
   open "http://localhost:8765/redesign_<page_name>.html"
   ```

3. Сообщи пользователю что макет открыт в браузере и опиши:
   - Краткое описание что было изменено по сравнению с текущей реализацией
   - Список использованных компонентов из дизайн-системы

4. Спроси пользователя через `AskUserQuestion`:
   - **Всё ли устраивает?** — Загрузить в Figma / Нужны правки
   - Если пользователь выбрал "Нужны правки" — уточни какие именно правки, внеси изменения в HTML-файл, обнови страницу в браузере (повторно `open` URL) и снова спроси

5. **Повторяй цикл правок** пока пользователь не одобрит макет

### Шаг 6 — Захватить в Figma (только после одобрения)

1. Вызови `generate_figma_design` с параметрами:
   - `outputMode`: `existingFile`
   - `fileKey`: `LkLr8e2DCFKQyLlnBjaTTw`
   - Если пользователь выбрал существующую страницу — передай `nodeId` этой страницы

2. Открой HTML в браузере с hash-параметрами захвата:
   ```bash
   open "http://localhost:8765/redesign_<page_name>.html#figmacapture=<captureId>&figmaendpoint=<endpoint>&figmadelay=3000"
   ```

3. Опрашивай статус каждые 5 секунд (до 10 раз):
   ```
   generate_figma_design(captureId: "<captureId>")
   ```

4. Останови сервер после завершения:
   ```bash
   kill <pid>
   ```

### Шаг 7 — Показать результат

Покажи пользователю:
- Ссылку на страницу в Figma
- Подтверждение что одобренный макет загружен

## Если нужного цвета/компонента нет в Figma

Если при создании макета обнаружилось, что в дизайн-системе отсутствует нужный цвет, стиль или компонент:

1. **Сообщи пользователю** — опиши что именно отсутствует
2. **Предложи добавить** — опиши какой токен/компонент нужен и с какими параметрами
3. **После согласования** — добавь недостающий элемент в Figma-файл дизайн-системы (через `generate_figma_design` на страницу `8:2`) и затем используй его в макете страницы

## Критические правила

- **Figma — единственный источник правды.** НЕ используй md-файлы документации для токенов/компонентов. Всегда бери актуальные значения из Figma через `get_design_context`
- **НЕ придумывай цвета** — если цвета нет в Figma, предложи добавить его туда
- **НЕ упрощай функциональность** — редизайн должен сохранять ВСЕ возможности текущей страницы
- **Используй Material Design** — стиль компонентов должен соответствовать Material Design 3
- **Русский язык** — все тексты в макете на русском
- **Реалистичные данные** — не используй Lorem Ipsum, заполняй правдоподобными данными из домена приложения (мафия, игроки, клубы, турниры)
- **Figma file key** — всегда добавляй в файл `LkLr8e2DCFKQyLlnBjaTTw`
