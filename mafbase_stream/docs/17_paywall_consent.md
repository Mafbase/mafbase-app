# Задача 17: Платный доступ и consent

## Цель

Стрим доступен только на оплаченных турнирах; перед стартом стримера обязуют согласиться на трансляцию изображения участников.

## Связь с BRD

- §4.7 F7.1–F7.3: фича только для оплаченных турниров, paywall при попытке запуска.
- §5.4 NF4.3: явный consent flow.
- §10 Q5: уточнить минимальную длительность турнира — может быть лимит.

## Содержание

1. На экране активного турнира кнопка «Запустить трансляцию» **видна всегда**, но при тапе на неоплаченный турнир — открывается экран с информацией об оплате (а не экран настройки).
2. Логика проверки `tournament.isPaid` уже есть в Mafbase (используем существующий флаг). Если модель сложнее (тарифные планы) — уточнить с продуктом и адаптировать.
3. Перед первым стартом стрима в данном турнире — диалог consent:
   - текст согласия (русский, согласован с юристами по §10 Q4 — параллельная задача);
   - чекбокс «Я подтверждаю, что получил согласие участников турнира на трансляцию имени, изображения и игровой роли»;
   - кнопка «Подтвердить» активна только при отмеченном чекбоксе.
4. Состояние согласия сохраняется на уровне турнира (в локальном хранилище и/или на бэкенде, чтобы не спрашивать повторно при перезапуске).
5. Без галочки — старт стрима невозможен.

## Критерий приёмки

- Турнир без оплаты: тап «Запустить трансляцию» → экран с информацией об оплате.
- Оплаченный турнир, первый запуск стрима: появляется consent-диалог. Без галочки кнопка disabled.
- Подтвердил → попадаешь на экран настройки (задача 15).
- Повторный запуск стрима в том же турнире — без consent (сохранено).
- Локализованные тексты в `app_ru.arb`.
- Golden-тесты диалога и paywall-экрана.

## Артефакты

- `lib/features/stream_paywall/` — экран информации об оплате.
- `lib/features/stream_consent/` — диалог согласия.
- Адаптер `StreamConsentRepository`.
- Тексты согласия — отдельный файл `assets/legal/stream_consent_ru.md` (для лёгкого обновления юристами).
