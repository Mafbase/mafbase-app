# Задача 10: Механизм рендеринга нативной view в видеопоток

## Цель

Реализовать в плагине универсальный механизм: приложение передаёт плагину произвольную **нативную view** (Android `android.view.View`, iOS `UIView`), плагин рендерит её в композит видеопотока поверх кадра камеры. Содержимое view на этом этапе значения не имеет — для проверки используется тестовая заглушка. Конкретная overlay-view с плашками игроков реализуется в задаче 11.

Идея: вёрстка оверлея ведётся отдельно от трансляции, в привычных для платформы инструментах (Compose / SwiftUI / XML / UIKit), без рендеринга текста и графики в C++.

## Связь с BRD

- §4.3.2 F3.6, F3.8, F3.9: оверлей всегда виден, не перерисовывается без изменений, поддерживает полупрозрачность.
- §1.3: оверлей собирается на устройстве до отправки в RTMP.

## Содержание

1. API в Dart:
   - `MafbaseStream.attachOverlayView(OverlayViewHandle handle)` — принимает идентификатор нативной view, зарегистрированной приложением через фабрику (по аналогии с PlatformView).
   - `MafbaseStream.detachOverlayView()`.
   - `MafbaseStream.invalidateOverlay()` — сигнал, что содержимое view изменилось и текстуру нужно перезалить.
2. Размер view фиксируется и равен размеру кадра видео (например 1920×1080). Фон view прозрачный (Android `setBackgroundColor(TRANSPARENT)`, iOS `isOpaque = false`).
3. Android-сторона (`OverlayViewRenderer.kt`):
   - получает `View` от приложения, делает `measure` + `layout` под размер кадра;
   - при `invalidate` рисует view в `Bitmap` через `view.draw(Canvas)` (или `PixelCopy`), загружает в GL-текстуру.
4. iOS-сторона (`OverlayViewRenderer.swift`):
   - получает `UIView`, выставляет `frame` под размер кадра;
   - при `invalidate` рендерит `CALayer.render(in:)` в `CGContext` поверх `CVPixelBuffer`, загружает в Metal/GL-текстуру.
5. Композит: alpha-blend полученной RGBA-текстуры поверх кадра камеры в FBO. Шейдер — простой `mix(camera, overlay, overlay.a)`.
6. Кеш: текстура пересоздаётся **только** по явному `invalidate` (F3.8). В простое — переиспользуется прошлый кадр оверлея.
7. В `example` — кнопка «Stream with test overlay», которая регистрирует тестовую view (полупрозрачный прямоугольник с текстом «TEST» посередине) и стартует стрим. Кнопка «Toggle overlay text» дёргает `invalidate`, чтобы проверить обновление.
8. Отладочный preview-режим: возможность запустить overlay view **без камеры и стриминга** — на экране устройства поверх однотонного фона (например серого), в реальном размере кадра. Нужен для независимой работы над вёрсткой оверлея в задаче 11. Реализуется в `example` отдельным экраном «Overlay preview», который монтирует ту же нативную view напрямую во Flutter через PlatformView, без участия стриминг-пайплайна.

## Критерий приёмки

- В VLC, подключённом к локальному nginx-rtmp, видно стрим с камеры с тестовой view поверх кадра.
- Текст в тестовой view читаемый, фон полупрозрачный.
- Нажатие «Toggle overlay text» в example меняет содержимое view в стриме в течение одного кадра.
- 30-секундный стрим без artifacts (текст не дрожит, плашки не «дёргаются»).
- Профайлер показывает, что без `invalidate` upload текстуры не происходит.
- В `example` есть экран «Overlay preview», на котором тестовая нативная view отрисовывается **без запуска стрима и камеры** — поверх однотонного фона, в размере кадра. Изменения вёрстки видны на этом экране без необходимости поднимать nginx-rtmp и подключать VLC.

## Артефакты

- `lib/src/overlay_view.dart` — Dart API и фабрика handle.
- `android/src/main/kotlin/.../OverlayViewRenderer.kt` — снятие View в Bitmap и upload в GL.
- `ios/Classes/OverlayViewRenderer.swift` — снятие UIView в pixel buffer и upload.
- Шейдер композита в `native/src/gl/shaders/overlay_composite.frag`.
- В `example/` — тестовая нативная view и UI для проверки.
