# Задача 14: Фоновое выполнение стрима

## Цель

Стрим продолжается, если приложение случайно ушло в фон (Home, lock screen) — насколько это разрешено платформой.

## Связь с BRD

- §4.6 F6.3: foreground service Android, background mode iOS.
- §5.2 NF2.3: корректная обработка входящих звонков.
- §5.1 NF1.4: устройство не перегревается за 1 час.
- §9 R5: iOS background-ограничения требуют коммуникации с пользователем.

## Содержание

### Android

1. `ForegroundService` с `FOREGROUND_SERVICE_TYPE_CAMERA | MICROPHONE | MEDIA_PROJECTION` (по необходимости).
2. Постоянная нотификация «Идёт трансляция • XX:YY» с кнопкой «Остановить».
3. Permissions для API 34+: `FOREGROUND_SERVICE_CAMERA`, `FOREGROUND_SERVICE_MICROPHONE`.
4. При уходе в фон стрим продолжается, в нотификации — длительность.

### iOS

1. `UIBackgroundModes`: `audio` для непрерывности аудиопотока.
2. Видеозахват в бэкграунде Apple запрещает — при сворачивании камеру выключаем, отправляем в RTMP **последний кадр + текстовая плашка «Стрим на паузе»**, аудио продолжается.
3. При возврате в foreground — захват камеры возобновляется.
4. Чёткое сообщение пользователю в UI: «Не сворачивайте приложение во время стрима» (см. задачу 16).

### Общие

5. Обработка входящего звонка (Android `TelephonyManager`, iOS `AVAudioSession.interruptionNotification`): аудио мьютится / приостанавливается, видео продолжается, после звонка — восстанавливается.
6. `Wake lock` / `keep-screen-on` пока сессия активна.

## Критерий приёмки

- Android: стрим запущен, нажат Home → в нотификациях видна постоянная нотификация, в VLC поток продолжается без обрывов. Тап по нотификации возвращает в Activity.
- iOS: стрим запущен, нажат Home → в VLC аудио продолжается, видео — статичный «paused»-кадр. Возврат в приложение → видео возобновляется.
- Входящий звонок на обеих платформах: стрим не падает, после звонка — восстанавливается.

## Артефакты

- `android/.../service/StreamForegroundService.kt`.
- `ios/Classes/Background/BackgroundCoordinator.swift`.
- Обновление `AndroidManifest.xml`, `Info.plist`.
