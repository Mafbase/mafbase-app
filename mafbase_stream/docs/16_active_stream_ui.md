# Задача 16: UI активного стрима

## Цель

Во время активной трансляции стримеру виден полноэкранный нативный экран с превью, статистикой и кнопками управления. После остановки — экран итогов.

## Связь с BRD

- §4.6 F6.1: превью, длительность, индикатор сети, битрейт, уровень микрофона, кнопки пауза/стоп/переключение камер.
- §4.6 F6.2: экран итогов после остановки.

## Содержание

1. На нативном экране стрима поверх композитного превью — Flutter-овый или нативный layer с UI:
   - таймер `HH:MM:SS`;
   - индикатор сети (зелёный/жёлтый/красный) на основе событий из задачи 13;
   - текущий битрейт (kbps);
   - индикатор уровня микрофона (RMS, обновление 10 Гц);
   - кнопки `CustomButton`: пауза/возобновление, стоп, переключение фронт↔тыл (BRD §4.6 F6.1).
2. Подписка на `EventChannel` плагина: `StreamEvent` → обновление UI.
3. После «Стоп» — экран итогов:
   - длительность;
   - средний битрейт;
   - путь к локальному файлу (если запись была);
   - кнопка «Поделиться» (share sheet);
   - кнопка «Закрыть» — возврат на экран турнира.
4. Подсказка для iOS «Не сворачивайте приложение» (см. задачу 14, §9 R5).
5. Все тексты — через локализацию `app_ru.arb`.

## Критерий приёмки

- Реальный турнир, реальный стрим: видны все индикаторы, цифры обновляются в реальном времени.
- Кнопка «Переключить камеру» меняет источник без обрыва стрима (зрители видят склейку).
- Кнопка «Пауза» приостанавливает видео, аудио продолжается, в VLC — «замороженный» кадр.
- Кнопка «Стоп» → экран итогов с корректными значениями. Share работает.
- Golden-тест экрана итогов.

## Артефакты

- `android/.../ui/StreamOverlayView.kt` или Flutter overlay через `FlutterView` поверх Activity.
- `ios/Classes/UI/StreamOverlayView.swift` или подключение Flutter overlay над `UIViewController`.
- `lib/features/active_stream/` — Bloc слушающий EventChannel.
- `lib/features/stream_summary/` — экран итогов.
