# Пользовательские флоу: Турнир

---

## Флоу 1: Плашки (OBS-виджеты для трансляции)

Полный путь организатора турнира от создания турнира до управления плашками во время стрима.

### Предусловия

- Пользователь авторизован в приложении

### Шаги

**1. Создание турнира**

1. Пользователь открывает список турниров (`/tournament`)
2. Нажимает кнопку «+» (FAB) для создания нового турнира
3. Турнир создаётся, пользователь попадает на страницу турнира (`/tournament/:id`)

**2. Добавление игроков**

4. В боковом меню (секция «Основное») пользователь нажимает «Добавить игрока»
5. Открывается диалог добавления игрока — пользователь вводит данные
6. Повторяет, пока все участники не добавлены

**3. Настройка турнира**

7. В боковом меню (секция «Управление») пользователь нажимает «Настройки»
8. На странице настроек (`/tournament/:id/settings`) задаёт:
   - Количество основных игр
   - Количество швейцарских игр (если нужно)
   - Количество финальных игр
   - Схему рейтинга
9. Нажимает «Сохранить»

**4. Оплата плашек**

10. В боковом меню (секция «Управление») пользователь нажимает «Оплата»
11. Открывается диалог оплаты с двумя параметрами:
    - **Количество игроков** — кратно 10, можно увеличить/уменьшить кнопками «+»/«−»
    - **Плашки** — чекбокс для включения OBS-виджетов
12. Пользователь ставит галочку «Плашки»
13. Отображается итоговая стоимость:
    - 10 игроков без плашек — бесплатно
    - 10 игроков + плашки — 300₽
    - 20+ игроков — 300₽ за каждые 10 без плашек, 400₽ за каждые 10 с плашками
14. Нажимает «Оплатить»
15. После успешной оплаты в боковом меню (секция «Функции») появляется пункт «Трансляция»

**5. Генерация или импорт рассадки**

> **Важно:** Без рассадки плашки будут пустыми — OBS-виджет не знает, какие игроки сидят за столом.

16. Пользователь переходит на страницу рассадки: в боковом меню (секция «Основное») нажимает «Рассадка»
17. Открывается страница рассадки (`/tournament/:id/editSeating`)
18. Пользователь выбирает один из способов:
    - **Генерация в Mafbase** — нажимает «Сгенерировать» для автоматической рассадки на основе добавленных игроков
    - **Импорт из GoMafia** — нажимает «Загрузить с Gomafia» и следует шагам из [Флоу 3: Импорт рассадки из GoMafia](#флоу-3-импорт-рассадки-из-gomafia)
19. После применения рассадки игроки распределены по столам, можно переходить к настройке плашек

**6. Получение ссылок для OBS**

20. В боковом меню пользователь раскрывает пункт «Трансляция» (иконка Live TV)
21. Открывается панель с параметрами:
    - **Дизайн** — кнопка выбора визуального оформления плашек
    - **Скопировать ссылки для всех столов** — кнопка копирования

**6а. Выбор дизайна (опционально)**

22. Нажимает на кнопку дизайна (по умолчанию — «mafbase»)
23. Открывается диалог выбора дизайна с превью каждого варианта
24. Выбирает понравившийся дизайн, нажимает «Выбрать»
25. Дизайн применяется, ссылки обновляются

**6б. Копирование ссылок**

26. Нажимает «Скопировать ссылки для всех столов»
27. В буфер обмена копируется текст со ссылками для каждого стола:
    ```
    Стол 1:
    Ссылка для встраивания в обс: https://mafbase.ru/translation?tournamentId=...&table=1&key=...&design=...
    Ссылка для управления: https://mafbase.ru/translationControl?tournamentId=...&table=1&key=...&design=...

    Стол 2:
    ...
    ```
    Количество столов = количество игроков ÷ 10
28. Появляется снэкбар «Ссылки скопированы»

**7. Настройка OBS**

29. Пользователь открывает OBS Studio
30. Добавляет источник «Браузер» (Browser Source)
31. Выставляет разрешение источника **1920×1080** — при другом разрешении виджет покажет предупреждение
32. Вставляет **ссылку для встраивания** (`/translation?...`) — это виджет, который видят зрители
33. Настраивает положение виджета на сцене

**8. Управление плашками во время игры**

Плашки можно использовать в двух режимах:

**8а. С ручным управлением**

33. Пользователь открывает **ссылку для управления** (`/translationControl?...`) в браузере
34. На странице управления:
    - **Выбор игры** — dropdown с номером текущей игры (от 1 до максимума)
    - **Список из 10 игроков** — для каждого игрока отображается:
      - Номер места (1–10)
      - **Роль** — переключатель роли (мирный, мафия, шериф, дон)
      - **Статус** — переключатель статуса (живой, убит, удалён и т.д.)
35. При выборе другой игры — данные обновляются автоматически
36. При изменении роли или статуса — изменения мгновенно отражаются на виджете в OBS через WebSocket-соединение

**8б. Без ручного управления**

37. Плашки можно использовать без открытия страницы управления
38. При отправке уведомления о начале следующей игры (см. [Флоу 2, шаг 6а](#флоу-2-уведомления-игрокам-на-турнире)) плашки автоматически переключаются на соответствующую игру — рассадка обновляется на виджете
39. В этом режиме на плашках отображаются только никнеймы игроков за столом, без ролей и статусов

### Результат

Зрители трансляции видят в реальном времени виджет с никнеймами игроков за столом. При ручном управлении — также роли и статусы. Плашки переключаются между играми автоматически при отправке уведомления о начале игры или вручную через страницу управления.

## Флоу 2: Уведомления игрокам на турнире

Полный путь организатора от создания турнира до отправки уведомлений участникам через Telegram и ВКонтакте.

### Каналы доставки уведомлений

Уведомления доставляются игрокам через два канала:

| Канал | Как игрок подключается | Условие получения |
|-------|----------------------|-------------------|
| **Telegram** | Игрок самостоятельно пишет свой ФСМ-ник боту [@mafbase_notification_bot](https://t.me/mafbase_notification_bot) | Игрок должен подписаться на бота и указать свой ФСМ-ник |
| **ВКонтакте** | Автоматически | Профиль ВК игрока должен быть привязан к его профилю GoMafia |

### Предусловия

- Организатор авторизован в приложении
- Турнир оплачен
- Для получения уведомлений игроки должны быть подключены хотя бы к одному каналу (см. таблицу выше)

### Шаги

**1. Создание турнира**

1. Пользователь открывает список турниров (`/tournament`)
2. Нажимает кнопку «+» (FAB) для создания нового турнира
3. Турнир создаётся, пользователь попадает на страницу турнира

**2. Добавление игроков**

4. В боковом меню нажимает «Добавить игрока»
5. Добавляет всех участников турнира

**3. Настройка количества игр**

6. В боковом меню переходит в «Настройки» (`/tournament/:id/settings`)
7. Указывает количество основных игр (например, 6)
8. Нажимает «Сохранить»

> **Важно:** Если количество игр = 0, при попытке отправить оповещение о начале игры отобразится предупреждение: «Укажите количество игр в настройках турнира»

**4. Оплата турнира**

9. В боковом меню нажимает «Оплата»
10. Указывает количество оплачиваемых игроков (кратно 10)
11. Нажимает «Оплатить»

**5. Секция «Оповещения» в меню**

12. После оплаты и при включённых уведомлениях в боковом меню появляется секция «Оповещения» с тремя пунктами:
    - **Оповещение о начале игры** (иконка колокольчика)
    - **Отправить текст** (иконка сообщения)
    - **Описания столов** (иконка местоположения)

**6а. Отправка оповещения о начале игры**

13. Пользователь раскрывает пункт «Оповещение о начале игры»
14. Открывается форма с двумя полями:
    - **Время** — кнопка для выбора времени через системный time picker (необязательно)
    - **Номер игры** — dropdown от 1 до максимального номера игры
15. Выбирает номер игры (обязательно) и, при желании, время начала
16. Нажимает «Отправить»
17. Участникам турнира отправляется уведомление о начале указанной игры через Telegram и/или ВК
18. Появляется снэкбар «Оповещение успешно отправлено»
19. Форма автоматически сворачивается

**6б. Отправка произвольного текста**

20. Пользователь раскрывает пункт «Отправить текст»
21. Открывается форма с текстовым полем (до 3 строк)
22. Вводит произвольное сообщение (например: «Перерыв 15 минут», «Сбор у стола 3»)
23. Кнопка «Отправить» становится активной (неактивна при пустом поле)
24. Нажимает «Отправить»
25. Участникам турнира отправляется уведомление с введённым текстом через Telegram и/или ВК
26. Появляется снэкбар «Оповещение успешно отправлено»
27. Форма автоматически сворачивается

**6в. Описания столов**

28. Пользователь нажимает «Описания столов»
29. Открывается страница `/tournament/:id/table-descriptions`
30. Для каждого стола можно указать текстовое описание его расположения (например: «2 этаж, комната у лестницы», «Зал B, левый угол»)
31. Эти описания включаются в уведомления о начале игры — игрок получает не только номер своего стола, но и информацию о том, где этот стол находится

### Типичные сценарии использования

| Сценарий | Тип уведомления | Пример |
|----------|-----------------|--------|
| Начинается 3-я игра в 15:30 | Начало игры | Игра 3, время 15:30 |
| Технический перерыв | Произвольный текст | «Перерыв 20 минут» |
| Смена локации | Произвольный текст | «Переезд в зал B» |
| Объявление результатов | Произвольный текст | «Результаты на стенде у входа» |

### Результат

Участники турнира получают уведомления через Telegram (если подписались на бота) и/или ВКонтакте (если профиль GoMafia связан с ФСМ). Организатор может оперативно информировать игроков о начале игр и любых организационных моментах.

### Подготовка игроков к получению уведомлений

Организатору рекомендуется заранее сообщить участникам:

1. **Telegram**: написать боту [@mafbase_notification_bot](https://t.me/mafbase_notification_bot) свой ФСМ-ник
2. **ВК**: убедиться, что профиль ВК привязан к профилю GoMafia

## Флоу 3: Импорт рассадки из GoMafia

Вместо генерации рассадки в Mafbase организатор может загрузить готовую рассадку из GoMafia (gomafia.pro) — стороннего сервиса управления турнирами по мафии.

### Предусловия

- Организатор авторизован в приложении
- Турнир создан и оплачен
- На GoMafia существует турнир с готовой рассадкой

> **Примечание:** Добавлять игроков в турнир заранее **не обязательно**. При импорте рассадки из GoMafia текущий список игроков турнира заменяется — все существующие игроки удаляются и добавляются игроки из импортированной рассадки.

### Шаги

**1. Переход на страницу рассадки**

1. В боковом меню (секция «Основное») пользователь нажимает «Рассадка»
2. Открывается страница рассадки (`/tournament/:id/editSeating`)

**2. Выбор опции «Загрузить с GoMafia»**

3. В меню действий (верхняя панель) пользователь нажимает «Загрузить с Gomafia»
4. Открывается диалог ввода URL турнира на GoMafia

**3. Ввод ссылки на турнир GoMafia**

5. Пользователь вводит URL турнира, например: `https://gomafia.pro/tournament/12345`
6. Если у турнира уже была привязана ссылка GoMafia — поле заполнено автоматически
7. Система валидирует формат ссылки (должен соответствовать `https://gomafia.pro/tournament/{id}`)
8. Пользователь подтверждает ввод

**4. Загрузка рассадки и импорт игроков**

9. Система отправляет запрос на сервер с ID турнира GoMafia
10. Сервер загружает рассадку из GoMafia
11. Текущие игроки турнира удаляются и заменяются игроками из рассадки GoMafia
12. Система сопоставляет импортированных игроков по ФСМ-никнеймам

**5а. Все игроки сопоставлены**

13. Если все игроки из рассадки GoMafia успешно сопоставлены по ФСМ-никнеймам:
    - Рассадка и список игроков автоматически применяются
    - Страница рассадки обновляется с новыми данными
    - Импорт завершён

**5б. Некоторые игроки не сопоставлены**

14. Если часть игроков из GoMafia не найдена по никнеймам в системе, открывается диалог сопоставления
15. Для каждого ненайденного игрока система показывает:
    - Текст: «Не найден игрок с никнеймом: {никнейм_с_гомафии}»
    - Подсказка: «Выберите игрока из списка, либо создайте нового»
16. Организатор для каждого ненайденного игрока выбирает одно из действий:
    - **Выбрать существующего** — привязать к существующему игроку в системе (его ФСМ-никнейм обновится для будущих импортов)
    - **Создать нового** — создать нового игрока с никнеймом из GoMafia
17. После сопоставления всех игроков рассадка применяется
18. Страница рассадки обновляется

### Когда использовать

| Сценарий | Рекомендация |
|----------|-------------|
| Турнир проходит в рамках ФСМ | Импорт из GoMafia — организатор обязан вести турнир на GoMafia, Mafbase является дополнением |
| Турнир не связан с ФСМ | Генерация рассадки в Mafbase — всё можно сделать на нашем сервисе |

### Результат

Рассадка из GoMafia загружена в Mafbase, список игроков турнира заменён на игроков из импортированной рассадки. При повторных импортах сопоставление уже не потребуется, так как ФСМ-никнеймы сохранены.
