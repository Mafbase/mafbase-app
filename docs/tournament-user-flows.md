# Пользовательские флоу: Турнир

---

## Флоу 1: Плашки (OBS-виджеты для трансляции)

Полный путь организатора турнира от создания турнира до управления плашками во время стрима.

### Предусловия

- Пользователь авторизован в приложении

### Шаги

**1. Создание турнира**

1. Пользователь открывает список турниров (`/tournament`)
2. Нажимает кнопку «+» (FAB) для создания нового турнира
3. Турнир создаётся, пользователь попадает на страницу турнира (`/tournament/:id`)

**2. Добавление игроков**

4. В боковом меню (секция «Основное») пользователь нажимает «Добавить игрока»
5. Открывается диалог добавления игрока — пользователь вводит данные
6. Повторяет, пока все участники не добавлены

**3. Настройка турнира**

7. В боковом меню (секция «Управление») пользователь нажимает «Настройки»
8. На странице настроек (`/tournament/:id/settings`) задаёт:
   - Количество основных игр
   - Количество швейцарских игр (если нужно)
   - Количество финальных игр
   - Схему рейтинга
9. Нажимает «Сохранить»

**4. Оплата плашек**

10. В боковом меню (секция «Управление») пользователь нажимает «Оплата»
11. Открывается диалог оплаты с двумя параметрами:
    - **Количество игроков** — кратно 10, можно увеличить/уменьшить кнопками «+»/«−»
    - **Плашки** — чекбокс для включения OBS-виджетов
12. Пользователь ставит галочку «Плашки»
13. Отображается итоговая стоимость:
    - 10 игроков без плашек — бесплатно
    - 10 игроков + плашки — 300₽
    - 20+ игроков — 300₽ за каждые 10 без плашек, 400₽ за каждые 10 с плашками
14. Нажимает «Оплатить»
15. После успешной оплаты в боковом меню (секция «Функции») появляется пункт «Трансляция»

**5. Получение ссылок для OBS**

16. В боковом меню пользователь раскрывает пункт «Трансляция» (иконка Live TV)
17. Открывается панель с параметрами:
    - **Дизайн** — кнопка выбора визуального оформления плашек
    - **Скопировать ссылки для всех столов** — кнопка копирования

**5а. Выбор дизайна (опционально)**

18. Нажимает на кнопку дизайна (по умолчанию — «mafbase»)
19. Открывается диалог выбора дизайна с превью каждого варианта
20. Выбирает понравившийся дизайн, нажимает «Выбрать»
21. Дизайн применяется, ссылки обновляются

**5б. Копирование ссылок**

22. Нажимает «Скопировать ссылки для всех столов»
23. В буфер обмена копируется текст со ссылками для каждого стола:
    ```
    Стол 1:
    Ссылка для встраивания в обс: https://mafbase.ru/translation?tournamentId=...&table=1&key=...&design=...
    Ссылка для управления: https://mafbase.ru/translationControl?tournamentId=...&table=1&key=...&design=...

    Стол 2:
    ...
    ```
    Количество столов = количество игроков ÷ 10
24. Появляется снэкбар «Ссылки скопированы»

**6. Настройка OBS**

25. Пользователь открывает OBS Studio
26. Добавляет источник «Браузер» (Browser Source)
27. Вставляет **ссылку для встраивания** (`/translation?...`) — это виджет, который видят зрители
28. Настраивает размер и положение виджета на сцене

**7. Управление плашками во время игры**

29. Пользователь открывает **ссылку для управления** (`/translationControl?...`) в браузере
30. На странице управления:
    - **Выбор игры** — dropdown с номером текущей игры (от 1 до максимума)
    - **Список из 10 игроков** — для каждого игрока отображается:
      - Номер места (1–10)
      - **Роль** — переключатель роли (мирный, мафия, шериф, дон)
      - **Статус** — переключатель статуса (живой, убит, удалён и т.д.)
31. При выборе другой игры — данные обновляются автоматически
32. При изменении роли или статуса — изменения мгновенно отражаются на виджете в OBS через WebSocket-соединение

### Результат

Зрители трансляции видят в реальном времени виджет с никнеймами, ролями и статусами игроков за столом. Организатор управляет отображением через отдельную страницу в браузере.

## Флоу 2: Уведомления игрокам на турнире

Полный путь организатора от создания турнира до отправки уведомлений участникам через Telegram и ВКонтакте.

### Каналы доставки уведомлений

Уведомления доставляются игрокам через два канала:

| Канал | Как игрок подключается | Условие получения |
|-------|----------------------|-------------------|
| **Telegram** | Игрок самостоятельно пишет свой ФСМ-ник боту [@mafbase_notification_bot](https://t.me/mafbase_notification_bot) | Игрок должен подписаться на бота и указать свой ФСМ-ник |
| **ВКонтакте** | Автоматически | Профиль игрока на GoMafia (сторонний сервис) должен быть связан с его ФСМ-профилем |

### Предусловия

- Организатор авторизован в приложении
- У турнира включены уведомления (флаг `notificationEnabled` устанавливается на стороне сервера)
- Для получения уведомлений игроки должны быть подключены хотя бы к одному каналу (см. таблицу выше)

### Шаги

**1. Создание турнира**

1. Пользователь открывает список турниров (`/tournament`)
2. Нажимает кнопку «+» (FAB) для создания нового турнира
3. Турнир создаётся, пользователь попадает на страницу турнира

**2. Добавление игроков**

4. В боковом меню нажимает «Добавить игрока»
5. Добавляет всех участников турнира

**3. Настройка количества игр**

6. В боковом меню переходит в «Настройки» (`/tournament/:id/settings`)
7. Указывает количество основных игр (например, 6)
8. Нажимает «Сохранить»

> **Важно:** Если количество игр = 0, при попытке отправить оповещение о начале игры отобразится предупреждение: «Укажите количество игр в настройках турнира»

**4. Оплата турнира**

9. В боковом меню нажимает «Оплата»
10. Указывает количество оплачиваемых игроков (кратно 10)
11. Нажимает «Оплатить»

**5. Секция «Оповещения» в меню**

12. После оплаты и при включённых уведомлениях в боковом меню появляется секция «Оповещения» с тремя пунктами:
    - **Оповещение о начале игры** (иконка колокольчика)
    - **Отправить текст** (иконка сообщения)
    - **Описания столов** (иконка местоположения)

**6а. Отправка оповещения о начале игры**

13. Пользователь раскрывает пункт «Оповещение о начале игры»
14. Открывается форма с двумя полями:
    - **Время** — кнопка для выбора времени через системный time picker (необязательно)
    - **Номер игры** — dropdown от 1 до максимального номера игры
15. Выбирает номер игры (обязательно) и, при желании, время начала
16. Нажимает «Отправить»
17. Участникам турнира отправляется уведомление о начале указанной игры через Telegram и/или ВК
18. Появляется снэкбар «Оповещение успешно отправлено»
19. Форма автоматически сворачивается

**6б. Отправка произвольного текста**

20. Пользователь раскрывает пункт «Отправить текст»
21. Открывается форма с текстовым полем (до 3 строк)
22. Вводит произвольное сообщение (например: «Перерыв 15 минут», «Сбор у стола 3»)
23. Кнопка «Отправить» становится активной (неактивна при пустом поле)
24. Нажимает «Отправить»
25. Участникам турнира отправляется уведомление с введённым текстом через Telegram и/или ВК
26. Появляется снэкбар «Оповещение успешно отправлено»
27. Форма автоматически сворачивается

**6в. Описания столов**

28. Пользователь нажимает «Описания столов»
29. Открывается страница `/tournament/:id/table-descriptions`
30. Здесь можно указать текстовые описания для каждого стола (например, расположение в помещении)
31. Эта информация отображается игрокам при просмотре рассадки

### Типичные сценарии использования

| Сценарий | Тип уведомления | Пример |
|----------|-----------------|--------|
| Начинается 3-я игра в 15:30 | Начало игры | Игра 3, время 15:30 |
| Технический перерыв | Произвольный текст | «Перерыв 20 минут» |
| Смена локации | Произвольный текст | «Переезд в зал B» |
| Объявление результатов | Произвольный текст | «Результаты на стенде у входа» |

### Результат

Участники турнира получают уведомления через Telegram (если подписались на бота) и/или ВКонтакте (если профиль GoMafia связан с ФСМ). Организатор может оперативно информировать игроков о начале игр и любых организационных моментах.

### Подготовка игроков к получению уведомлений

Организатору рекомендуется заранее сообщить участникам:

1. **Telegram**: написать боту [@mafbase_notification_bot](https://t.me/mafbase_notification_bot) свой ФСМ-ник
2. **ВК**: убедиться, что профиль GoMafia привязан к ФСМ-аккаунту
